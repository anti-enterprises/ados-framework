# Tasks

## ADOS Task Rules

- Use `- [ ] T001` task rows so ADOS can parse `tasks.md`.
- Mark independent tasks with `[P]`.
- Mark user-story work with `[US1]`, `[US2]`, etc.
- Put exact file paths in backticks.
- Put tests before implementation when behavior changes.

## Example

- [ ] T001 Create failing unit test in `tests/example.test.ts`
- [ ] T002 [P] [US1] Add API contract test in `tests/api.contract.test.ts`
- [ ] T003 [US1] Implement API handler in `src/api.ts`
- [ ] T004 [US1] Verify implementation with `npm test`
